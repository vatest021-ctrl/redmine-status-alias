# Redmine Status Alias

Плагин для Redmine 6.0.x, который показывает клиентам выбранные внешние названия статусов обращений без изменения внутренних статусов, workflow и истории задач.

## Назначение

В технической поддержке внутренние статусы Redmine часто слишком детальны для клиентов. `redmine_status_alias` позволяет администратору выбрать роли клиентов и настроить соответствие:

- внутренний статус Redmine;
- статус, который должен видеть клиент.

Например, внутренний статус `Диагностика у 2 линии` можно показывать клиенту как `В работе`.

## Совместимость

- Redmine: `6.0.0+`, проверочный целевой стенд: `6.0.5`
- Ruby/Rails: используются версии из установленного Redmine
- Миграции БД: не требуются

## Установка

Скопируйте или клонируйте репозиторий в каталог плагинов Redmine:

```bash
cd /path/to/redmine/plugins
git clone git@github.com:vatest021-ctrl/redmine-status-alias.git redmine_status_alias
```

Перезапустите Redmine. Миграции можно не запускать, но безопасно выполнить стандартную команду:

```bash
cd /path/to/redmine
bundle exec rails redmine:plugins:migrate RAILS_ENV=production
```

## Настройка

Откройте:

```text
Администрирование -> Модули -> Redmine Status Alias -> Настроить
```

Далее:

1. включите плагин;
2. выберите роли клиентов;
3. задайте соответствие внутренних статусов статусам, видимым клиенту;
4. сохраните настройки.

Администраторы всегда видят исходные внутренние статусы.

## Как это работает

Плагин подключает небольшой patch к `IssueStatus#name`. Если текущий пользователь имеет одну из выбранных клиентских ролей, то вместо имени внутреннего статуса возвращается имя выбранного alias-статуса. ID статуса задачи не меняется.

Это означает:

- workflow Redmine не меняется;
- отчеты и фильтры продолжают работать по исходным статусам;
- история задач остается технически корректной;
- клиентский интерфейс показывает более понятные статусы.

## Ограничение текущей версии

Проверка роли выполняется по всем членствам пользователя в Redmine. Практически это подходит для выделенных клиентских учетных записей. Если один и тот же пользователь одновременно является клиентом в одном проекте и внутренним сотрудником в другом, ему будут показываться alias-статусы во всех проектах.
