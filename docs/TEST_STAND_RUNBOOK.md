# Test Stand Runbook

## Быстрое подключение на стенде

```bash
cd /Users/valeriypopov/redmine_status_alias
sudo scripts/link_to_redmine.sh /home/red2mine/20240627/red2mine
sudo scripts/migrate_plugin.sh /home/red2mine/20240627/red2mine
```

После этого перезапустите Rails/Passenger/Puma-процесс Redmine.

## Проверка

1. Войдите администратором.
2. Откройте настройки плагина `Redmine Status Alias`.
3. Выберите клиентскую роль.
4. Для нескольких внутренних статусов выберите клиентские alias-статусы.
5. Откройте нужный проект.
6. Включите модуль `Клиентские алиасы статусов` на вкладке `Настройки -> Модули`.
7. Войдите пользователем с выбранной ролью в этом проекте.
8. Откройте список задач и карточку задачи.
9. Убедитесь, что клиент видит alias-статусы в проекте с включенным модулем.
10. Откройте проект без включенного модуля и убедитесь, что там отображаются исходные статусы.
11. Войдите администратором и убедитесь, что администратор видит исходные статусы.
