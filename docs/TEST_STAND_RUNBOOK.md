# Test Stand Runbook

## Быстрое подключение на стенде

```bash
cd /Users/valeriypopov/redmine_status_alias
sudo scripts/link_to_redmine.sh /home/red2mine/20240627/red2mine
sudo scripts/migrate_plugin.sh /home/red2mine/20240627/red2mine
```

После этого перезапустите Rails/Passenger/Puma-процесс Redmine.

## Проверка

1. Войдите администратором.
2. Откройте `Администрирование -> Роли и права доступа`.
3. Выдайте клиентской роли право `Видеть клиентские алиасы статусов`.
4. Откройте настройки плагина `Redmine Status Alias`.
5. Убедитесь, что в списке клиентских ролей доступны только роли с правом клиентского просмотра.
6. Выберите клиентскую роль.
7. При необходимости выберите служебную роль, которая не должна видеть alias-статусы.
8. Для нескольких внутренних статусов выберите клиентские alias-статусы.
9. Если один клиентский alias назначен нескольким внутренним статусам, настройте блок `Клиентские переходы`.
10. Откройте нужный проект.
11. Включите модуль `Клиентские алиасы статусов` на вкладке `Настройки -> Модули`.
12. Войдите пользователем с выбранной клиентской ролью в этом проекте.
13. Откройте список задач и карточку задачи.
14. Убедитесь, что клиент видит alias-статусы в проекте с включенным модулем.
15. Откройте форму редактирования задачи и контекстное меню, проверьте названия доступных статусов.
16. Если доступен неоднозначный alias, убедитесь, что в форме показан один клиентский пункт и при сохранении выставляется target из `Клиентских переходов`.
17. Измените статус задачей клиентского пользователя и проверьте историю изменений.
18. Проверьте email-уведомление о смене статуса.
19. Проверьте фильтр списка задач по статусу и проектный отчет по задачам.
20. Откройте проект без включенного модуля и убедитесь, что там отображаются исходные статусы.
21. Войдите администратором и убедитесь, что администратор видит исходные статусы.
