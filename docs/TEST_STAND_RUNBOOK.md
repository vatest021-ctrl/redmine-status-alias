# Test Stand Runbook

## Быстрое подключение на стенде

```bash
cd /Users/valeriypopov/redmine_status_alias
sudo scripts/link_to_redmine.sh /home/red2mine/20240627/red2mine
sudo scripts/migrate_plugin.sh /home/red2mine/20240627/red2mine
```

После этого перезапустите Rails/Passenger/Puma-процесс Redmine.

## Проверка

1. Войдите администратором.
2. Откройте настройки плагина `Redmine Status Alias`.
3. Выберите клиентскую роль.
4. При необходимости выберите служебную роль, которая не должна видеть alias-статусы.
5. Для нескольких внутренних статусов выберите клиентские alias-статусы.
6. Откройте нужный проект.
7. Включите модуль `Клиентские алиасы статусов` на вкладке `Настройки -> Модули`.
8. Войдите пользователем с выбранной клиентской ролью в этом проекте.
9. Откройте список задач и карточку задачи.
10. Убедитесь, что клиент видит alias-статусы в проекте с включенным модулем.
11. Откройте форму редактирования задачи и контекстное меню, проверьте названия доступных статусов.
12. Измените статус задачей клиентского пользователя и проверьте историю изменений.
13. Проверьте email-уведомление о смене статуса.
14. Проверьте фильтр списка задач по статусу и проектный отчет по задачам.
15. Откройте проект без включенного модуля и убедитесь, что там отображаются исходные статусы.
16. Войдите администратором и убедитесь, что администратор видит исходные статусы.
