# Test Stand Runbook

## Быстрое подключение на стенде

```bash
cd /Users/valeriypopov/redmine_status_alias
sudo scripts/link_to_redmine.sh /home/red2mine/20240627/red2mine
sudo scripts/migrate_plugin.sh /home/red2mine/20240627/red2mine
```

После этого перезапустите Rails/Passenger/Puma-процесс Redmine.

## Проверка

1. Войдите администратором.
2. Откройте настройки плагина `Redmine Status Alias`.
3. Выберите клиентскую роль.
4. При необходимости выберите служебную роль, которая не должна видеть alias-статусы.
5. Оставьте включенной настройку `Применять алиасы для каналов без получателя`, если групповые боты отправляют сообщения клиентам.
6. Для нескольких внутренних статусов выберите клиентские alias-статусы.
7. Откройте нужный проект.
8. Включите модуль `Клиентские алиасы статусов` на вкладке `Настройки -> Модули`.
9. Войдите пользователем с выбранной клиентской ролью в этом проекте.
10. Откройте список задач и карточку задачи.
11. Убедитесь, что клиент видит alias-статусы в проекте с включенным модулем.
12. Откройте форму редактирования задачи и контекстное меню, проверьте названия доступных статусов.
13. Измените статус задачей клиентского пользователя и проверьте историю изменений.
14. Проверьте email-уведомление о смене статуса.
15. Проверьте InTouch live-уведомление в личный Telegram/Matrix/MAX/Slack-канал клиента.
16. Проверьте InTouch group-уведомление в клиентский групповой чат.
17. Проверьте регулярное InTouch-уведомление.
18. Проверьте фильтр списка задач по статусу и проектный отчет по задачам.
19. Откройте проект без включенного модуля и убедитесь, что там отображаются исходные статусы.
20. Войдите администратором и убедитесь, что администратор видит исходные статусы.
